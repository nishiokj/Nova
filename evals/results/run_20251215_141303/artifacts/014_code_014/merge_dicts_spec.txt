merge_dicts specification

Function signature:
    def merge_dicts(d1: dict, d2: dict) -> dict

Behavior:
- Returns a new dictionary containing the union of keys from d1 and d2.
- For keys that appear only in one dictionary, the resulting dictionary will contain that key with its original value.
- For keys that appear in both dictionaries, the resulting dictionary will contain the key with the sum of the two values.
  - The function will attempt to add the two values using the + operator. If addition succeeds, the summed value is used.
  - If addition raises an exception (e.g., values are incompatible types), merge_dicts will raise a TypeError with an informative message.
- The input dictionaries are not modified (function is pure with respect to inputs).
- Keys must be hashable (as required by Python dicts).

Example:
    merge_dicts({'a': 1, 'b': 2}, {'b': 3, 'c': 4}) -> {'a': 1, 'b': 5, 'c': 4}

Complexity:
- Time: O(n + m) where n and m are the number of items in d1 and d2, respectively.
- Space: O(n + m) for the new dictionary.

Notes / design choices:
- Rather than restricting values strictly to numbers, the implementation will attempt to use +, which allows numeric types and other types that define addition (e.g., Decimals). This is flexible but will raise a TypeError if addition is unsupported.
- If you prefer a strict numeric-only function, require values to be instances of numbers.Number and raise a TypeError otherwise.
