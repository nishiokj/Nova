# Agent Execution Logging System

## Overview

The agent execution logging system provides comprehensive, structured tracking of agent behavior, planning, and execution. It creates a dedicated log file (`agent_execution.jsonl`) that captures:

1. **Full Request Context** - Complete context presented to the agent
2. **Planning Results** - Detailed plans with all substeps and success criteria
3. **Execution Context** - Context provided to each subagent/substep
4. **Execution Step Logs** - Granular step results with failure modes

## Architecture

### New Component: `harness/agent_execution_logger.py`

A dedicated logger that outputs to `logs/agent_execution.jsonl` in JSONL format (one JSON object per line).

**Key Features:**
- Separate from general request/health logs for focused analysis
- All entries include standard metadata: `ts`, `lvl`, `svc`, `req_id`, `exec_id`
- Execution ID (`exec_id`) links all logs for a single agent execution
- Single-line JSON format for easy parsing and analysis

### Log Entry Types

#### 1. Agent Context Log (`svc: "agent_context"`)

Captures the complete context presented to the agent at request start.

**Fields:**
```json
{
  "ts": "2025-12-10T14:56:32.039Z",
  "lvl": "INFO",
  "svc": "agent_context",
  "req_id": "abc123-0001",
  "exec_id": "abc123-0001-exec-0001",
  "context": {
    "tier": "standard",
    "system_prompt": "You are a capable assistant...",
    "conversation_history": [
      {"role": "user", "content": "Previous message..."},
      {"role": "assistant", "content": "Previous response..."}
    ],
    "tool_definitions": [
      {"name": "fast_answer", "description": "Search and answer..."}
    ],
    "tool_count": 12,
    "additional": {
      "context_provided": "Optional additional context"
    }
  },
  "request": {
    "user_input": "What is the weather today?",
    "input_length": 25,
    "has_context": true
  }
}
```

**Purpose:** Understand exactly what information the agent had when making decisions.

#### 2. Planning Result Log (`svc: "planning"`)

Captures the complete execution plan generated by the Planner.

**Fields:**
```json
{
  "ts": "2025-12-10T14:56:32.145Z",
  "lvl": "INFO",
  "svc": "planning",
  "req_id": "abc123-0001",
  "exec_id": "abc123-0001-exec-0001",
  "plan": {
    "goal": "Find current weather information",
    "goal_type": "search",
    "requires_tools": true,
    "estimated_complexity": "standard",
    "success_criteria": "User gets accurate, current weather information",
    "reasoning": "Needs current/external information",
    "status": "pending",
    "steps": [
      {
        "step_num": 1,
        "objective": "Search for current weather",
        "tool_hint": "fast_answer",
        "tool_args_hint": {"query": "current weather today"},
        "success_criteria": "Find relevant, current information",
        "depends_on": [],
        "status": "pending"
      },
      {
        "step_num": 2,
        "objective": "Synthesize findings into answer",
        "tool_hint": null,
        "success_criteria": "Provide clear answer based on search results",
        "depends_on": [1],
        "status": "pending"
      }
    ],
    "step_count": 2
  }
}
```

**Purpose:** Understand the agent's strategy and planned approach before execution.

#### 3. Execution Context Log (`svc: "execution_context"`)

Captures the context provided to each substep/subagent during execution.

**Fields:**
```json
{
  "ts": "2025-12-10T14:56:32.250Z",
  "lvl": "INFO",
  "svc": "execution_context",
  "req_id": "abc123-0001",
  "exec_id": "abc123-0001-exec-0001",
  "step_id": "step-1",
  "context": {
    "step_num": 1,
    "step_objective": "Search for current weather",
    "tool_hint": "fast_answer",
    "message_count": 2,
    "messages": [
      {"role": "system", "content": "You are a capable assistant..."},
      {"role": "user", "content": "What is the weather today?"}
    ],
    "available_tools": ["fast_answer", "web_search", "calculator"],
    "dependencies": []
  }
}
```

**Purpose:** Understand what each substep received for context during execution.

#### 4. Execution Step Log (`svc: "execution_step"`)

Captures detailed results for each execution step, including failure modes.

**Success Example:**
```json
{
  "ts": "2025-12-10T14:56:32.500Z",
  "lvl": "INFO",
  "svc": "execution_step",
  "req_id": "abc123-0001",
  "exec_id": "abc123-0001-exec-0001",
  "step_id": "step-1",
  "step_num": 1,
  "status": "completed",
  "result": {
    "step_objective": "Search for current weather",
    "duration_ms": 234,
    "tool_used": "fast_answer",
    "tool_result": "Current temperature is 72°F with partly cloudy skies...",
    "result_truncated": false
  }
}
```

**Failure Example:**
```json
{
  "ts": "2025-12-10T14:56:32.500Z",
  "lvl": "WARNING",
  "svc": "execution_step",
  "req_id": "abc123-0001",
  "exec_id": "abc123-0001-exec-0001",
  "step_id": "step-2",
  "step_num": 2,
  "status": "failed",
  "result": {
    "step_objective": "Execute file operation",
    "duration_ms": 50,
    "tool_used": "file_write"
  },
  "failure_mode": "permission_error",
  "substeps_failed": ["Write to protected directory"],
  "error_details": {
    "error_message": "Permission denied: /etc/config.json",
    "failure_mode": "permission_error"
  }
}
```

**Failure Modes Tracked:**
- `tool_disabled` - Tool is not enabled
- `timeout` - Tool execution exceeded timeout
- `permission_error` - Permission/access denied
- `tool_execution_failed` - General tool failure

**Purpose:** Detailed forensics of what succeeded/failed and why.

## Integration Points

### 1. Agent (`harness/agent.py`)

**Initialization:**
```python
self.exec_logger = get_execution_logger()
```

**In `Agent.run()` method:**
- Generates `exec_id` from `req_id`
- Logs full agent context before planning
- Logs planning result after plan creation
- Passes `req_id` and `exec_id` to executor

### 2. Planner (`harness/planner.py`)

**Initialization:**
```python
self.exec_logger = get_execution_logger()
```

**In `Planner.create_plan()` method:**
- Accepts optional `exec_id` parameter (not used internally yet)

### 3. Executor (`harness/planner.py`)

**In `Executor.execute()` method:**
- Accepts `req_id` and `exec_id` parameters
- Logs execution context before each substep
- Logs detailed execution step results after each tool call
- Tracks failure modes and categorizes errors

## Log File Format

**File:** `logs/agent_execution.jsonl`

**Format:** JSONL (JSON Lines) - one JSON object per line

**Size:** Up to 50MB with 10 backup files (configurable)

**Example Log Sequence for a Single Request:**
```jsonl
{"ts":"2025-12-10T14:56:32.039Z","lvl":"INFO","svc":"agent_context",...}
{"ts":"2025-12-10T14:56:32.145Z","lvl":"INFO","svc":"planning",...}
{"ts":"2025-12-10T14:56:32.250Z","lvl":"INFO","svc":"execution_context",...}
{"ts":"2025-12-10T14:56:32.500Z","lvl":"INFO","svc":"execution_step",...}
{"ts":"2025-12-10T14:56:32.750Z","lvl":"INFO","svc":"execution_step",...}
```

## Usage Examples

### Parsing Logs

```python
import json
from pathlib import Path

log_file = Path("logs/agent_execution.jsonl")

# Read all entries
with open(log_file, 'r') as f:
    entries = [json.loads(line) for line in f]

# Filter by service type
context_logs = [e for e in entries if e['svc'] == 'agent_context']
planning_logs = [e for e in entries if e['svc'] == 'planning']
step_logs = [e for e in entries if e['svc'] == 'execution_step']

# Filter by execution ID
exec_id = "abc123-0001-exec-0001"
execution_logs = [e for e in entries if e.get('exec_id') == exec_id]

# Find failures
failures = [e for e in entries if e.get('status') == 'failed']
```

### Analysis Queries

**Find all failed executions:**
```bash
grep '"status": "failed"' logs/agent_execution.jsonl | jq .
```

**Get all plans for a specific request:**
```bash
grep '"req_id": "abc123-0001"' logs/agent_execution.jsonl | grep '"svc": "planning"' | jq .
```

**Count failure modes:**
```bash
grep '"failure_mode"' logs/agent_execution.jsonl | jq -r '.failure_mode' | sort | uniq -c
```

**View execution timeline for an exec_id:**
```bash
grep '"exec_id": "abc123-0001-exec-0001"' logs/agent_execution.jsonl | jq '.ts, .svc, .status // empty'
```

## Relationship to Other Logs

The system maintains **three separate log files**:

1. **`requests.jsonl`** - High-level request lifecycle (INFO)
   - User-facing events
   - Clean, readable summary of request flow
   - Suitable for monitoring and alerting

2. **`health.jsonl`** - System health and diagnostics (DEBUG)
   - Heartbeats, VAD events, TTS status
   - Internal system metrics
   - Performance and health monitoring

3. **`agent_execution.jsonl`** - Detailed agent execution (INFO/WARNING) **[NEW]**
   - Full agent context and decisions
   - Complete planning and execution details
   - Granular step-by-step forensics
   - Suitable for debugging and analysis

All three logs share the same `req_id` for correlation.

## Configuration

The execution logger can be configured in `harness/agent_execution_logger.py`:

```python
logger = AgentExecutionLogger(
    log_dir="logs",                      # Log directory
    max_log_size=50 * 1024 * 1024,      # 50MB per file
    backup_count=10                      # Keep 10 backup files
)
```

Total storage: ~500MB (50MB × 10 files + current)
